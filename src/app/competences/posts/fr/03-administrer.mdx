## Niveau 2 - Déployer des services dans une architecture

### Concevoir et développer des applications communicantes

**Description :** Créer des applications distribuées qui communiquent via des protocoles réseau.

**Ce que j'ai fait dans l'Application de Monitoring Linux :**
- Développé un protocole de communication custom basé sur JSON sur TCP
- Implémenté un système client-serveur pour centraliser les métriques de plusieurs machines
- Créé des APIs REST pour exposer les données de monitoring aux applications externes
- Mis en place un système de heartbeat pour détecter les machines déconnectées
- Configuré la communication bidirectionnelle pour les commandes de contrôle

**Technologies utilisées :** Python sockets,FastAPI

### Utiliser des serveurs et des services réseaux virtualisés

**Description :** Déployer et gérer des services dans un environnement virtualisé ou containerisé.

**Ce que j'ai fait dans l'Application de Monitoring Linux :**
- Containerisé l'application avec Docker pour faciliter le déploiement
- Configuré docker-compose pour orchestrer les services (monitoring agent, collecteur central, base de données)
- Testé le monitoring sur des machines virtuelles avec VMware et VirtualBox
- Configuré les volumes Docker pour persister les logs et configurations
- Implémenté la surveillance des conteneurs Docker et de leurs ressources

**Technologies utilisées :** Docker, docker-compose, monitoring de conteneurs, volumes persistants,
