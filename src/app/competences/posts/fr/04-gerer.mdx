### Concevoir une base de données relationnelle à partir d'un cahier des charges

**Description :** Dans la SAÉ1.04 Création d'une base de données, nous avons conçu une base de données en fonction d'un cahier des charges.

**Ce que j'ai fait dans l'Application de Prêt Étudiant :**
- Analysé le cahier des charges : gestion des étudiants, objets empruntables, prêts et retours
- Créé le modèle conceptuel avec diagramme entité-relation (Étudiant, Objet, Prêt, Catégorie)
- Normalisé la base jusqu'à la 3ème forme normale pour éviter la redondance
- Défini les contraintes d'intégrité : clés primaires, étrangères, contraintes de check
- Implémenté le schéma physique avec optimisation des types de données (DATE, DECIMAL, ENUM)

**Technologies utilisées :** Modélisation conceptuelle, MySQL, normalisation de base de données, contraintes d'intégrité

## Niveau 2 - Optimiser une base de données, interagir avec une application et mettre en œuvre la sécurité

### Optimiser les modèles de données de l'entreprise

**Description :** Améliorer les performances et l'efficacité des bases de données existantes.

**Ce que j'ai fait dans l'Application de Prêt Étudiant :**
- Analysé les performances avec EXPLAIN pour identifier les requêtes lentes
- Créé des index composites sur (etudiant_id, date_pret) pour optimiser les recherches
- Implémenté une table de cache pour les statistiques fréquemment consultées
- Optimisé les requêtes de rapport en utilisant des procédures stockées
- Ajouté une table d'historique pour archiver les anciens prêts et améliorer les performances

**Technologies utilisées :** Optimisation MySQL, index composites, procédures stockées, mise en cache, archivage de données

### Manipuler des données hétérogènes

**Description :** Intégrer et traiter des données provenant de sources multiples et de formats différents.

**Ce que j'ai fait dans le ChatBot AI avec Ollama :**
- Traité des flux audio en temps réel pour la reconnaissance vocale (Speech-to-Text API)
- Converti les données audio en texte puis en prompts pour le LLM Ollama
- Intégré les réponses textuelles du LLM avec l'API ElevenLabs pour la synthèse vocale

- Synchronisé les flux de données asynchrones entre WebSockets, LLM local et APIs externes
- Optimisé les conversions de formats pour minimiser la latence du chatbot

**Technologies utilisées :** Audio processing, Speech-to-Text API, LLM Ollama, ElevenLabs TTS, WebSockets, format conversion
